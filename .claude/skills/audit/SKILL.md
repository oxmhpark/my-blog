# 수행할 작업

## 환경설정

- `_config.yml` 파일에서 컬렉션 별 소스 폴더와 uri 체계를 확인할 것.

## 패스 1：프론트매터 기반 연결 검사

- `/_entry_**/` 폴더 하위의 포스트 파일들을 순회하면서 프론트매터에 정의된 분류 페이지가 실제로 존재하는지 확인한다.
- `/_archive_**/` 폴더 하위의 분류 페이지 파일들을 순회하면서 포스트와 연결되었는지 확인한다.

### 분류 페이지 유형과 경로

| 프론트매터 키 | 프론트매터 형식 | 분류 페이지 경로 | 비고 |
|---|---|---|---|
| `projects`| 배열 | `/_archive_projects/` | 커스텀 프론트매터 |
| `events` | 배열 |`/_archive_events/` | 커스텀 프론트매터 |
| `years` | 배열 |`/_archive_years/` | 커스텀 프론트매터 |

### 분류 페이지 매칭 규칙

- 공백과 특수문자를 포함해 문자열 전체를 원형 그대로 비교한다.
- 프론트매터 값과 확장자를 제외한 파일 이름이 완전히 일치해야 한다.
- 대소문자를 구분한다.

## 패스 2：본문·요약 기반 내부 링크 연결 검사

- 내부 링크는 스키마, 프로토콜, 도메인 이름이 없는 절대 경로(`/링크-url`) 또는 상대 경로(`링크-url`) 형식을 가리킨다.
- 전체 엔트리의 본문 및 `excerpt` 프론트매터 문자열에 포함된 내부 연결이 실제로 존재하는 페이지를 가리키는지 점검한다.
- 위키 마크업과 HTML 마크업을 모두 검사한다 (예: `[링크 텍스트](/링크-url)` 또는 `<a href="/링크-url">링크 텍스트</a>`)
- `<img>` `<video>` `<audio>` 태그의 `src` 속성에 포함된 내부 연결도 검사한다.
- Liquid 템플릿 문법이 적용된 링크는 검사하지 않는다. (예: `{{ site.baseurl }}/링크-url`)
- 앵커·쿼리에서 `#` `?` 뒤에 오는 부분은 무시한다. (예: `/링크-url#앵커`, `/링크-url?쿼리스트링`)

## 패스 3：보고서 작성

- 검출된 항목을 표 형식의 보고서로 작성해서 `/.audits/YYYY-MM-DD_HH-mm-ss.md` 경로에 저장한다. (UTF-8, GMT+9)

### 보고서 형식

표 구성:

| 컬럼 | 정렬 | 설명 | 비고 |
|---|---|---|---|
| File | ASC | 검출 파일 | 검사한 파일 경로 |
| Line | ASC | 검출 행 | 고립된 분류인 경우 0 |
| Message | ASC(코드 포인트 기준) | 메시지 | 후술 |

메시지 타입:

| 유형 | 형식 | 비고 |
|---|---|---|
| `projects` 프론트매터에서 존재하지 않는 분류 페이지 검출 | `중분류 깨짐: {중분류 slug}` |  |
| `events` 프론트매터에서 존재하지 않는 분류 페이지 검출 | `소분류 깨짐: {소분류 slug}` |  |
| `years` 프론트매터에서 존재하지 않는 분류 페이지 검출 | `연도 분류 깨짐: {연도 분류 slug}` |  |
| 본문 및 `excerpt` 프론트매터 문자열에서 존재하지 않는 내부 링크 검출 | `내부 링크 깨짐: {링크 url}` |  |
| 고립된 분류 | `고립된 분류` |  |

### 보고서 가독성

보고서를 렌더링 없이도 쉽게 읽을 수 있도록 다음과 같은 규칙을 적용한다:

- 각 컬럼의 너비를 최장 셀에 맞춰 공백으로 패딩한다. (멀티바이트 문자는 2칸으로 계산)
- 구분선(---)도 컬럼 너비에 맞춰 늘린다.
- 빈 셀도 폭을 맞춘다.
- 패딩은 우측에 추가한다.

# 예외처리 규칙

- 포스트 초안(확장자를 제외한 파일 제목이 `_`로 시작하는 경우)도 검사한다.
- 템플릿 파일(확장자를 제외한 파일 제목이 `_`로 시작하고 `_`로 끝나는 경우)은 무시한다.
